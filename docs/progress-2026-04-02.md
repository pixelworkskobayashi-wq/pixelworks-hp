## 完成済み機能

### ヘッダー
- 固定表示（z-index: 100）
- PIXEL WORKS ロゴ（アクセントカラー）
- ナビ4項目：SERVICES・WORKSはトップページ内スクロール、ABOUT・CONTACTはページ遷移

### HERO
- フルスクリーン（100svh）
- Unsplash建築写真（後で実写真に差し替え可能）
- グラデーションオーバーレイ
- キャッチコピー＋サブテキスト
- CTAリンク：`VIEW WORKS →` / `CONTACT`

### SERVICES
- 5サービスをグリッド表示（04「画像合成・加工」を削除、01〜05に採番）
- 番号（01〜05）＋日本語タイトル＋英語サブタイトル＋説明文（任意）
- スクロールフェードイン

### WORKS
- **作品数**: 24件（rendering/movie/vr/photo）
- **サムネ形状**: 横長（16:9）・縦長（3:4）・正方形（1:1）が混在
- **登場アニメーション**: IntersectionObserver
- **ホバー**: scale(1.06) ズーム
- **カテゴリフィルター**: ALL / RENDERING / MOVIE / VR / PHOTO
- **モーダル（画像）**: FLIP アニメーション、中間拡大・最大拡大・パン対応
- **モーダル（動画/VR）**: iframe モーダル（白背景）、YouTube・Twinmotion対応
- **サムネバッジ**: ▶（movie）/ VR（vr）
- **左右ナビ**: ボタン・キー・ドラッグ対応（flyImg上のみ）、画像↔動画/VR間の遷移も対応
- `embedUrl` フィールドで動画/VR切り替え（未設定時は画像モーダル）

## 作品画像の運用

| 環境 | 画像の場所 | URLの形式 |
|------|-----------|---------|
| ローカル開発 | `D:\Pixel Works\HP2026\works\` | `/works/ファイル名.jpg` |
| 本番（Netlify） | `public/works/` にコピーしてgit push | `/works/ファイル名.jpg` |

`astro.config.mjs` にViteミドルウェアを設定済みのため、ローカルではDドライブの画像をそのまま参照可能。

### works.ts への追加方法

```typescript
{ 
  id: 'work-xx',
  title: '作品タイトル',
  category: 'rendering', // rendering / movie / vr / photo
  shape: 'landscape',    // landscape / portrait / square
  imageUrl: '/works/ファイル名.jpg',
  description: '説明文（空文字列''でも可）',
  // 動画・VRの場合のみ追加:
  embedUrl: 'https://www.youtube.com/embed/VIDEO_ID',
  // または
  embedUrl: 'https://twinmotion.unrealengine.com/panorama/XXXX?embed',
}
```
